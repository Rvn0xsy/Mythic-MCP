# Changelog

All notable changes to this plugin will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-01-24

### Added
- Initial release of CI-First Development plugin
- **CI-First Philosophy Skill**: Guides systematic reasoning, integration-first testing, and CI validation
- **Integration Test Planning Skill**: Designs comprehensive test strategies with Docker orchestration and test data management
- **GitHub Actions Setup Skill**: Implements optimized CI/CD pipelines with caching and parallelization
- **CI Architect Agent**: Specialized assistant for CI/CD architecture and troubleshooting
- Complete documentation with real-world patterns from successful SDK development
- EnsureXExists pattern for test data management
- Phase-based test organization framework
- Rapid CI feedback loop methodology

### Philosophy
- Shifts from unit-test-driven TDD to integration and E2E testing with real services
- Treats CI as source of truth
- Emphasizes reasoning before coding
- Prioritizes comprehensive integration coverage over isolated unit tests
